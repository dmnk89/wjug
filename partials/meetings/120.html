O prezentacji:

Programowanie sieciowe w Javie od poczatku bylo jej kluczowym elementem. Gdy jest konieczna komunikacja miedzy systemami to rozwiazania takie jak REST, SOAP czy JMS najczesciej pasuja idealnie. Sa jednak problemy, gdzie ten poziom rozwiazan nie jest odpowiedni, np.:

  • integracje istniejacych systemow, stworzonych w roznych technologiach, czesto przed era RESTa, Javy
  • specyficzne protokoly dla danej domeny systemu
  • rozwiazania infrastruktury, programowanie systemowe

W takich sytuacjach konieczne jest uzycie narzedzi i API nizszego poziomu - gniazdek. I choc rowniez tutaj Java dostarcza dobrego wsparcia to jednak glebsza wiedza jest potrzebna, aby tworzone rozwiazanie bylo poprawne, bez ukrytych problemow, o ktore dosc latwo.

Celem prezentacji jest pokazanie mozliwosci Javy w programowaniu sieciowym na poziomie gniazdek poprzez omowienie:

  • istotnych API i mozliwych opcji
  • kluczowych elementow protokolu TCP i wynikajacych zagrozen
  • przykladu implementacji wlasnego protokolu warstwy aplikacji
  • roznych modeli obslugi IO: blokujacy, nieblokujacy, asynchroniczny
  • wybranych szczegolow implementacyjnych na poziomie systemu operacyjnego
  • ograniczen Javy w programowaniu systemowym
  • propozycji architektury

O prelegencie:
